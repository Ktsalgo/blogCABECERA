git init - abrir el area de preparacion (staging) y repositorio local   
git add - ayuda asigarnle un rastreo a un archivo para saber si ese archivo esta teniendo cambios 
git commit - envia todo lo que esta staging al repositorio
git commit -m "mensaje" = nos sirve para hacer carga masiva despues del GIT add
Git add . = nos sirve para cargar todo 
git commit -am "mensaje" = nos sirve para hacer la carga masiva sin tener que hacer git add


cuuando se trabaja con un equipo de trabajo uno tiene que tener un repositorio remoto donde todo el mundo puede interactuar 
git clone - va copiar todo el directorio de trabajo y repositorio local para llevarlo al repositorio  remoto.
git push - copia del repositorio local para alimentar el repositorio remoto
git fetch - trae una copia del repositorio remoto y la copia en el repositorio local. pero no la copia en el directorio de trabajo.
git merge - es para hacer una copia de repositorio local al directorio de trabajo
git pull - es el que nos hacer una copia del remoto para alimentar el directorio de trabajo y repositorio local

git log = para revisar como se ha hecho cada cambio y los COD para buscar 
git log --stat = mejor forma para visualizar cambios hechos 
git show 

git reset "COD" --soft = es para volver a la version anterior por lo que esta en Staging se mantiene en el directorio de trabajo vuelve a la anterior
git reset "COD" --hard = vuelve todo a la version que estamos se√±alando 

git diff = ayuda a comparar el directorio vs el staging

git checkout "COD" + nombre archivo = me trae el cambio para establecerlo ne la rama principal ( master  )

GIT merge = tengo que estar parado en donde quiero  que vaya las copias y convocar el origen del que vamos a copiar 

Git remote add "nombre de la conexion" "code que sacamos de git hub" = nos conecta y le da acceso a github para la sincronizacion
git remote = nos sirve para saber cuales son los nomnbres de las conexiones
git remote -v = nos informa los fetch y los push

git pull + nombre archivo + main = extraer de github al computador 
git pull origin main --allow-unrelated-histories = forzar la conexion de git hub y computador para extraer 
git push + nombre de la conexion + ...
